## 拡張機能名

ブックマーク受信箱

## 短い説明（132文字以内、一覧・検索結果に表示される）

フォルダ選択なしでページを一旦保存し、未処理件数をバッジ表示。「あとで読む」を分類の手間なく管理。

（52文字。日本語は1文字1バイト超のため、念のためダッシュボード上で文字数警告が出ないか確認すること）

## 詳細な説明

```
「あとで読もう」と思ったページ、ブックマークのフォルダ分けが面倒で結局
タブを開きっぱなしにしていませんか？

ブックマーク受信箱は、フォルダを選ばず一旦保存するだけの「受信箱」形式の
拡張機能です。あとでまとめて確認し、処理済みにするだけのシンプルな運用が
できます。

■ 主な機能
・ポップアップの「現在のページを保存」ボタン、または右クリックメニュー
　（ページ／リンク）で、分類せずに一旦保存
・未処理件数をツールバーバッジに常時表示
・「処理済み」にしても削除せず、状態変更のみで後から見返せる
・保存件数500件・容量4MiBの安全上限つき

■ プライバシーについて
・閲覧履歴やページ内容を外部に送信することは一切ありません
・すべての処理はブラウザ内で完結します
・保存したページのタイトル・URLは、この拡張機能の機能のためだけに
　お使いの端末内（chrome.storage.local）に保存されます

■ こんな方におすすめ
・「あとで読む」ページが増えすぎて、開きっぱなしのタブが多い方
・ブックマークのフォルダ分けを都度考えるのが面倒な方
・とりあえず保存だけして、あとでまとめて整理したい方

■ ご利用にあたっての注意
・未処理件数のバッジは、ツールバーアイコンをピン留めしていないと
　気づきにくい場合があります。アドレスバー右側のパズルピースアイコンから
　ピン留めすることをおすすめします
```

## 単一目的の説明（Single purpose、審査時に求められる場合がある）

このアプリケーションの唯一の目的は、ユーザーが選択したページを分類なしで
保存し、未処理件数を表示・管理することです。それ以外の機能（データ収集、
広告表示、外部送信等）は一切含みません。

## 権限の正当化理由（Permission justification）

### `storage`

保存したページのタイトル・URL・処理状態を`chrome.storage.local`に保存する
ために必要です。ブラウザ内にのみ保存され、外部サーバーへ送信されることは
ありません。

### `activeTab`

保存ボタンを押した時点でアクティブなタブのタイトル・URLを取得するために
必要です。ユーザーが明示的に保存操作をした場合にのみ使用します。

### `contextMenus`

右クリックメニューからページ・リンクを保存できるようにするために必要です。

## スクリーンショット（要手動撮影）

Chrome Web Storeは1280x800または640x400のスクリーンショットを推奨（1〜5枚）。
以下を撮影することを推奨：

1. 未処理一覧に複数件保存されているポップアップ画面
2. 右クリックメニューから保存する操作の画面
3. ツールバーのバッジに件数が表示されている状態（ピン留め済みアイコン）
4. 処理済みにした後の一覧画面

撮影方法：`chrome://extensions`で拡張機能を読み込んだ状態で、実際にページを
保存・処理済みにする操作をしながらスクリーンショットを撮る。
